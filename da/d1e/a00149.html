Detailed Description

Implementation of the java.lang.Runnable for the thread that is responsible for communication with each client.

The actions performed by this thread are :

  • Authenticate the client.
  • According to the result
    • If the client is not authentic, close the connection and terminate.
    • Otherwise, confirm the connection and start communication with the client.
  • Examine the data received from client and transfer it to a controller (project.pc.device.MouseController or project.pc.device.KeyboardController) to execute the input operations.

Client authentication is done by comparing the pairing key received from the client with the one displayed on the pairing key dialog window.

It receives data from client and wraps it in a DataWrapper object. It, then, invokes the DataWrapper#getOperationType() method to examine the type of operation requested by the client and accordingly transfers the control to either project.pc.device.MouseController or project.pc.device.KeyboardController.

This thread is started by the Server#listen() upon accepting every client connection. It gets terminated implicitly after unsuccessful authentication of the client. Explicitly, it is terminated by the NetworkManager (NetworkManager#stopServer() and NetworkManager#disconnect(InetAddress)) by invoking the setStopFlag() method at the time of server stop or disconnection of this client A reference of this thread gets stored in the NetworkState after successful authentication of the client.
